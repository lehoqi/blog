# 完整方案：恐龙 + 专属座驾个性化系统

## 一、玩家身份：恐龙 + 座驾组合

保留恐龙大框架，每个玩家拥有专属座驾：

| 玩家 | 恐龙 | 座驾 | 组合头像 | 身份标签 |
|------|------|------|----------|----------|
| 乐乐 | 🦕 | 🚓 | 🦕🚓 | 警车恐龙 |
| 昊昊 | 🦖 | 🚑 | 🦖🚑 | 急救恐龙 |

**改动点**：
- `PLAYERS` 对象新增 `vehicle` 和 `vehicleName` 字段
- 玩家选择页卡片头像改为组合展示（恐龙 + 座驾），下方加小标签如"🚓 警车迷"
- 答题页 player indicator、进度条恐龙都用组合头像
- 题目卡片内的小恐龙也改为当前玩家的组合头像

## 二、偏好加权出题

根据当前玩家，调整题目抽取概率：

- **有玩家时**：60% 概率抽取该玩家偏好主题的故事，40% 概率随机抽取通用故事
- **无玩家时**：完全随机（保持现有逻辑）

**实现方式**：
- `STORIES_ADD` 和 `STORIES_SUB` 中每个故事新增一个可选 `tag` 字段：`'police'`、`'ambulance'` 或不填（通用）
- `PLAYERS` 对象新增 `preferTag` 字段（乐乐 = `'police'`，昊昊 = `'ambulance'`）
- `generateQuestion()` 接收 `currentPlayer` 参数，按概率筛选故事

## 三、主题庆祝反馈

### 3.1 个性化庆祝语

答对题目后，根据故事的 `tag` 和当前玩家，显示个性化庆祝语：

**答对时**（`submitAnswer` 中 `showCelebrate` 之前）：
- 如果是警车题 + 乐乐："🚓 警笛响起！嘀嘟嘀嘟～破案成功！"
- 如果是救护车题 + 昊昊："🚑 急救成功！你是小英雄！"
- 如果是对方主题或通用题：用现有通用庆祝语

**语音也对应调整**：
- 警车题答对：speak("嘀嘟嘀嘟，太厉害了！")
- 救护车题答对：speak("急救成功，你真棒！")

### 3.2 全屏飞车动效（答对主题题目时触发）

当玩家答对**自己偏好主题**的题目时（乐乐答对警车题 / 昊昊答对救护车题），触发全屏飞车特效，类似直播送礼物的"跑车飞过屏幕"效果。

#### 视觉效果

1. **飞车主体（三段式节奏：快-慢-快）**：
   - **冲入阶段（0-500ms）**：车辆 emoji（🚓 或 🚑）从屏幕左侧外快速冲入，带有轻微倾斜（`rotate(-3deg)`），营造速度感
   - **中央亮相（500-800ms）**：车辆到达屏幕中央时减速，放大至最大尺寸（约 `scale(1.4)`），短暂停顿 0.3 秒。此刻警灯闪光最强烈、警笛声最响——这是整个动画的"高光时刻"
   - **加速冲出（800-1400ms）**：车辆从中央加速冲出屏幕右侧，尺寸回缩，带有微微上扬角度（`rotate(2deg)`），像"起飞"一样消失
2. **星星尾迹**：车辆身后拖着一串 ⭐✨ 星星粒子，像彗星尾巴一样逐渐变小、渐隐消散。中央停顿时星星在车辆周围短暂"爆开"散落
3. **全屏警灯闪光**：车辆经过时，屏幕背景短暂闪烁对应颜色，模拟警灯打在墙上的效果
   - 警车🚓 = 红蓝交替闪烁（`#f44336` ↔ `#1565c0`）
   - 救护车🚑 = 红白交替闪烁（`#f44336` ↔ `#ffffff`）
   - 中央停顿时闪烁频率最高（约 100ms 一次），冲入和冲出阶段频率较低（约 200ms 一次）
4. **文字渐隐**：车飞过后，屏幕中央留下一行大字渐隐消失："🚓 破案成功！" 或 "🚑 急救成功！"

#### 音效

使用 Web Audio API（`AudioContext` + `OscillatorNode`）合成警笛/救护车音效，无需加载外部音频文件：

- **警车警笛**：双音交替（约 600Hz ↔ 800Hz），快速切换 3-4 次，模拟"嘀嘟嘀嘟"，持续约 1.2 秒
- **救护车警笛**：缓慢上下滑音（约 600Hz → 900Hz → 600Hz），模拟欧式救护车"嗡——呜——"声，持续约 1.5 秒

两种音效均使用正弦波（`sine`）振荡器 + `gain` 节点控制音量淡入淡出，确保不刺耳、对小孩友好。

#### 动画时序

| 时间 | 阶段 | 事件 |
|------|------|------|
| 0ms | 开始 | 创建浮层（`position: fixed; z-index: 9999`），开始播放警笛/救护车音效 |
| 0-100ms | 开始 | 全屏闪光开始（低频闪烁，200ms 一次） |
| 0-500ms | 冲入 | 车辆从左侧外快速冲入至屏幕中央，`easing: ease-out`，带 `rotate(-3deg)` |
| 0-500ms | 冲入 | 星星尾迹从车辆左侧持续喷出 |
| 500-800ms | 亮相 | 车辆在屏幕中央停顿，放大至 `scale(1.4)`，轻微上下浮动 |
| 500-800ms | 亮相 | 全屏闪光加速（高频闪烁，100ms 一次），警笛音量达到最大 |
| 500-800ms | 亮相 | 星星在车辆周围"爆开"散落 |
| 800-1400ms | 冲出 | 车辆加速冲出屏幕右侧，`easing: ease-in`，带 `rotate(2deg)` + 缩小 |
| 800-1400ms | 冲出 | 全屏闪光减弱并停止 |
| 1000ms | 文字 | 屏幕中央文字淡入（"🚓 破案成功！" / "🚑 急救成功！"） |
| 1400ms | 收尾 | 车辆完全消失，星星尾迹最后几颗渐隐 |
| 1500ms | 收尾 | 音效淡出结束 |
| 2000ms | 收尾 | 中央文字淡出 |
| 2200ms | 结束 | 移除浮层 DOM，继续进入下一题 |

#### 技术实现

- **DOM 结构**：一个 `position: fixed` 的全屏 `<div>` 浮层，内含车辆元素、星星粒子、闪光背景层、文字层
- **动画**：纯 CSS `@keyframes` + `element.animate()`（Web Animations API），与现有庆祝动画风格一致
- **音效**：`AudioContext` + `OscillatorNode`，封装为 `playPoliceSiren()` 和 `playAmbulanceSiren()` 两个函数
- **触发条件**：仅当 `currentPlayer` 的 `preferTag` 与当前题目的 `tag` 匹配时触发，避免过于频繁
- **性能**：纯 CSS 动画 + 少量 DOM 操作，不使用 canvas，低端手机也能流畅运行

#### 降级处理

- 如果浏览器不支持 `AudioContext`，静默跳过音效，视觉动画照常播放
- 如果用户设备开启了 `prefers-reduced-motion`，跳过飞车动画，仅显示文字庆祝

## 六、数据存储扩展

当前 `saveRecord` 只存 `{ player, score, total, time }`，需要扩展：

```javascript
{
  player: 'lele',
  score: 8,
  total: 10,
  time: 1710000000000,
  tagScores: { police: 3, ambulance: 1, general: 4 }
}
```

新增 `tagScores` 字段，记录每局各主题的答对数，用于解锁主题勋章。旧记录没有该字段时兼容处理即可。
